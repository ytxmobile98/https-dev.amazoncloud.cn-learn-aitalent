### Amazon Kendra 有何用途？

Amazon Kendra 是一项自然语言搜索服务，利用机器学习 (ML) 来改善搜索和用户体验。作为一项完全托管式服务，Amazon Kendra 包括一个开箱即用的预训练 ML 模型。客户之所以通常选择 Amazon Kendra，是因为它提高了搜索结果的准确性和搜索非结构化数据的能力。

借助 Amazon Kendra，您可以将多个数据存储库连接到索引并摄取和网络爬取文档，从而打造统一的搜索体验。您可以使用文档元数据为用户打造功能丰富的定制搜索体验，帮助他们有效地找到正确的查询答案。

### Amazon Kendra 可以解决什么问题？

企业中大约 80% 的数据都是非结构化数据，例如文档、帮助网站、支持文档和电子邮件。这些数据位于不同的孤岛（电子邮件、文档管理系统、Wiki 内容、工作流），并且通常是分散的，无法集中访问。

这可能会导致员工的工作效率降低、工作重复和创新停滞。客户可能难以搜索网站，从而导致客户不满。联系中心的代理也可能难以快速找到客户问题的答案。没有一个单一地方可以询问能够理解的问题，因为答案必须跨孤岛获取。

### Amazon Kendra 有哪些优势？

* **提高员工的工作效率**

    员工需要快速获取最新信息才能营造创新环境。Amazon Kendra 可以打造统一的搜索和发现体验，即使内容分散在多个存储库中也一样。它不仅可以为员工提供完成工作所需的信息，还能提高员工能力，并自动执行繁琐的任务。
* **加快研究和上市时间**

    通过更快、更准确的搜索，员工不再浪费时间搜索正确的信息，可以在一开始就富有成效。他们可以快速确定研究中断的位置和参与人员。然后，他们可以组建合适的团队，及时推动公司的下一波创新。
* **提高联系中心的效率，降低呼叫中心的成本**

    基于 ML 的智能搜索有助于解读客户的问题，并提供最相关的信息来解决他们的问题。
* **增强客户体验，提高采用率和保留率**

    为了打造具有持久忠诚度的企业，客户需要快速找到他们想要的内容，例如产品、公司退货政策的详细信息或者相关新闻文章。
* **改善 SaaS 中的搜索和客户体验**

    独立软件供应商 (ISV) 和数字原生企业是构建软件即服务 (SaaS) 应用程序的企业之一。许多 SaaS 应用程序包括关键字搜索，这可能会导致效率低下。

    使用 Amazon Kendra 和自然语言理解功能替换或增强搜索功能可以提高 SaaS 应用程序的工作效率和体验。
* **跨不同数据源整合信息**

    Amazon Kendra 提供 40 多个连接器以及 ISV 提供的几十个连接器。这意味着单个搜索可以跨整个企业。您可以整合来自电子邮件、文档管理系统、票证系统和 Wiki 等多个源的信息。

### Amazon Kendra 如何收费？

Amazon Kendra 的定价由两个主要驱动因素组成：存储和查询。存储取决于要编制索引的文档的数量。查询是指每天的查询总数。为了使所有客户都可以访问 Amazon Kendra，亚马逊云科技产品不仅可以满足小型部署的需求，还能满足大型企业组织的需求。这两个选项包括开发人员版 (KDE) 和企业版 (KEE)。

### 如何使用 Amazon Kendra 构建云解决方案？

Amazon Kendra 是一项智能企业搜索服务，可以帮助您使用内置连接器在内容存储库之间进行搜索。

![构建云解决方案](<./构建云解决方案.png>)

1. **搜索用户**

    用户与内联网搜索、网站搜索、自助服务机器人或联系中心等应用程序交互。
2. **Amazon Kendra**

    Amazon Kendra 使用自然语言理解 (NLU) 从其索引中获取结果。
3. **连接器**

    索引由跨越多个后端数据源的内置或自定义连接器填充。
4. **搜索结果**

    Amazon Kendra 返回搜索结果和按相关性排序的内容的链接。

### Amazon Kendra 有哪些基本技术概念？

Amazon Kendra 是一项高度准确的智能搜索服务，可以帮助您的用户使用自然语言处理和高级算法搜索非结构化和结构化数据。它会返回问题的具体答案，为用户提供一种接近于与人类专家互动的体验。Amazon Kendra 还可以与 Amazon Simple Storage Service (Amazon S3) 和 Amazon Lex 等其他亚马逊云科技服务紧密集成。

* **事实性问题**：一个类别是关于谁、什么、何时或何地的问题。事实性问题具有基于事实的答案，这些答案会以单个字词或短语的形式返回。系统将从常见问题或编制索引的文档中检索此类问题的答案。
* **描述性问题**：描述性问题是指答案可以是一个句子、一段话或一整个文档的问题。
* **关键字和自然语言搜索**：关键字和自然语言搜索使用的问题不仅包括要搜索的关键字，还包括复杂的对话式问题。这包括预期含义不一定明确的问题。Amazon Kendra 会推断用户搜索查询背后的预期含义，从而返回相关信息。该服务使用深度学习模型来处理此类查询。

### 如何使用生成式人工智能 (AI) 增强 Amazon Kendra 返回的搜索结果？

您可以使用检索增强型生成 (RAG) 来增强用户体验并提供利用大型语言模型 (LLM) 功能的生成式 AI 响应。搜索查询和结果提供 LLM 使用的上下文来获得生成式 AI 响应。

您可以使用 Amazon Kendra 索引从 Wiki 页面、Microsoft SharePoint 网站、Atlassian Confluence 和文档存储库（例如 Amazon S3）等数据源摄取企业非结构化数据。

使用 RAG 方法的生成式 AI 应用程序的架构：

![使用 RAG 方法的生成式 AI 应用程序的架构](<./使用RAG方法的生成式AI应用程序的架构.png>)

1. 用户向生成式 AI 应用程序发出请求。
2. 该应用程序根据用户请求向 Amazon Kendra 索引发出搜索查询。
3. 该索引返回搜索结果，其中包含所摄取企业数据中的相关文档的摘录。
4. 该应用程序将用户请求以及从索引中检索到的数据作为上下文发送到 LLM 提示符中。
5. LLM 根据检索到的数据向用户请求返回简洁的响应。
6. 系统将来自 LLM 的响应发送给用户。

借助该架构，您可以为您的使用案例选择最合适的 LLM。LLM 选项包括合作伙伴 Hugging Face、AI21 Labs、Cohere 和托管在 Amazon SageMaker 终端节点上的其他合作伙伴。其他选项包括 Anthropic 和 OpenAI 等公司的模型。

借助 Amazon Bedrock，您可以选择 Amazon Titan、Amazon LLM 或合作伙伴 LLM。合作伙伴 LLM 的示例包括 AI21 Labs 和 Anthropic，它们可以通过 Bedrock API 安全地进行访问，而您的数据无需离开亚马逊云科技。Amazon Bedrock 还提供其他一些优势，包括无服务器架构、用于调用受支持的 LLM 的单一 API，以及可以简化开发人员工作流的托管服务。

### Amazon Kendra 有哪些典型使用案例？

* **企业搜索**：帮助员工通过单一搜索界面发现最新的内容和问题的答案。这打破了数据孤岛，有助于提高员工的工作效率，营造创新环境。
* **Contact Center Intelligence**：使用 Amazon Kendra，通过直观的自助服务机器人、代理协助的解决方案和顺畅的文档访问，降低联系中心的成本。
* **SaaS 中的嵌套搜索**：改善应用程序中的搜索体验，通过由机器学习提供支持的应用程序内搜索更快地查找信息。

### 关于 Amazon Kendra，您还应该记住什么？

* **为安全性而设计**

    Amazon Kendra 的功能可用于根据用户访问权限管理对文档的访问。按组安排内容，确保只有授权用户才能访问相关文档。
* **创建数据源列表**

    聚合来自文档管理系统、电子邮件和 SaaS 产品等源的信息。
* **使用开箱即用的连接器**

    Amazon Kendra 具有一个不断扩展的列表，其中包含 30 多个内置连接器以及 100 多个由我们的合作伙伴提供的连接器，我们会经常添加更多连接器。
* **使用正规语言询问问题**

    例如，如果您想查找检验实验室的位置，可以询问我可以在哪里找到检验实验室？而不是仅仅输入检验实验室。 Amazon Kendra 将使用其自然语言功能来理解问题和内容，然后返回相关答案。
* **安排定期同步**

    经常同步可以帮助用户访问最新的数据。
